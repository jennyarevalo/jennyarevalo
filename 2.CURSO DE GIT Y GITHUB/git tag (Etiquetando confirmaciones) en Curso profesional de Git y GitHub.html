git tag (Etiquetando confirmaciones)

En este momento quizás ya quieras taggear o etiquetar los cambios. Para eso existe git tag, este simplemente solo va a agregar una etiqueta.
Una cosa a destacar es que son ligeras o anotadas, las ligeras se crean con el nombre de la etiqueta, las anotadas son las que llevan además un mensaje usando -m podemos dejar un mensaje.
Si queremos borrar una etiqueta debemos usar -d, si las queremos listar podemos usar -l y si las queremos renombrar usamos -f.
Utilizando el Sha-1 podemos crear etiquetas anotadas haciendo referencia a commits viejos.

Forzar la actualización de una etiqueta:

$ git tag-f -a 0.4 -m “Versión certificada” fb7f1b44

Nota: El atributo -f es aplicable cuando intentamos crear una etiqueta que ya ha sido creada y vinculada a un commit específico. Y podría ser utilizado para 2 operaciones:

  1. En caso necesitemos actualizar su mensaje (Para esto debemos referenciar al mismo código SHA del commit al que pertenece actualmente).
  2. En caso necesitemos vincular el tag a otro commit (El tag será desvinculado del commit original y vinculado al nuevo).

Importante: No es recomendable modificar un tag, si el commit asociado está siendo utilizado por otro usuario (Se aplicó Push / Pull). Revisar documentación (Sección DISCUSSION):

$ git tag--help

git tag –d [etiqueta] Para etiquetar versiones anteriores,
git tag –f –a [etiqueta] –m “[mensaje]” [sha-1] Para renombrar una etiqueta.

git tag

Crear etiquetas en git para versionar los repositorios
$ git tag -a [Numero de version] -m "[Comentario del tag]"
ejemplo: $ git tag -a 0.5 -m “Versión estable del proyecto”

Para ver la lista de tags que tenemos guardados en el repositorio se usa el comando $ git tag -l

Si en un caso ocupanos asignar un tag a un commit pasado usamos la erramienta de git log para ver los codigos sha-1 delos repositorios
despues de esto copiamos el sha del commit que deseamos asignar un tag usamos el siguiente comando
$ git tag [##] [codigo Sha-1 del commit que deseamos tagear]

ej: $ git tag 0.3 b6a90758fdc5f911c12ed7ece4a2b01a326f43ed

Para borrar etiquetas se utiliza el comando
$ git tag -d [version a borrar]
ej: $ git tag -d 0.5

Para renombrar es con
$ git tag -f [version a renombrar] -a [version corregida] -m “[comentario para el nuevo tag]” [Numero de Sha del commit]
ej: $ git tag -f 1.0 -a 0.8 -m “Este archivo remplaza los titulos” b6a90758fdc5f911c12ed7ece4a2b01a326f43ed
